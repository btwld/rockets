What changed, in one paragraph

defineResource() takes the minimum information you actually need to describe a CRUD resource — a repository key and an entity class — plus optional DTOs, joins, hooks, and handler overrides. It returns the same RocketsResourceConfig the core already consumes, so nothing downstream changes. The 50-line hand-written factory collapses to ~15 declarative lines; duplications of the resource name (seven places before) collapse to one.

pet.resource.ts — before and after

BEFORE — 50 lines 
import { ApiBearerAuth, ApiTags } from '@nestjs/swagger';
import { UseHooks } from '@bitwild/rockets-common';
import {
  CrudOperationResolver, CrudListQuery, CrudReadQuery,
  CrudCreateCommand, CrudUpdateCommand, CrudDeleteCommand,
  CrudJoin, Operation,
} from '@bitwild/rockets-crud';
import { OwnerScopeHook, type RocketsResourceConfig }
  from '@bitwild/rockets-core';
import { PetCreateDto, PetUpdateDto, PetResponseDto,
         PetPaginatedDto } from './pet.dto';
import { PetCreateHandler } from './pet-create.handler';

export function createPetResource(): RocketsResourceConfig {
  return {
    crud: {
      controller: {
        path: 'pets',
        entity: 'pet',
        resolver: CrudOperationResolver,
        response: { resource: PetResponseDto,
                    paginated: PetPaginatedDto },
        extraDecorators: [ApiBearerAuth(), ApiTags('Pets'),
                          UseHooks(OwnerScopeHook)],
      },
      operations: [
        { operation: Operation.List, query: CrudListQuery,
          extraDecorators: [CrudJoin([{ relation: 'vaccinations' }])] },
        { operation: Operation.Read, query: CrudReadQuery,
          extraDecorators: [CrudJoin([{ relation: 'vaccinations' }])] },
        { operation: Operation.Create,
          request: { body: PetCreateDto },
          command: CrudCreateCommand,
          commandHandler: PetCreateHandler },
        { operation: Operation.Update,
          request: { body: PetUpdateDto },
          command: CrudUpdateCommand },
        { operation: Operation.Delete,
          command: CrudDeleteCommand },
      ],
    },
    providers: [PetCreateHandler, OwnerScopeHook],
  };
}
AFTER — 15 lines 
import { OwnerScopeHook, defineResource }
  from '@bitwild/rockets-core';
import { PetEntity } from './pet.entity';
import { PetCreateDto, PetUpdateDto, PetResponseDto }
  from './pet.dto';
import { PetCreateHandler } from './pet-create.handler';

export const petResource = defineResource({
  key:       'pet',
  entity:    PetEntity,
  dto: {
    response: PetResponseDto,
    create:   PetCreateDto,
    update:   PetUpdateDto,
  },
  joins:    [{ relation: 'vaccinations' }],
  hooks:    [OwnerScopeHook],
  handlers: { create: PetCreateHandler },
  providers:[PetCreateHandler, OwnerScopeHook],
});
What disappeared:
  • PetPaginatedDto — auto-generated from PetResponseDto
  • path: 'pets' — pluralized from key
  • ApiTags('Pets') — derived from key
  • ApiBearerAuth() — default on
  • CrudOperationResolver — default resolver
  • Per-op query/command — auto-selected by Operation
  • Per-op CrudJoin — derived from top-level joins

RocketsResourceDefinition<E> — the complete surface

export interface RocketsResourceDefinition<E extends PlainLiteralObject> {
  /** Repository key. Also used to derive route path and tag. */
  readonly key: string;

  /** Entity class — drives type safety and (future) auto-registration. */
  readonly entity: Type<E>;

  /** DTOs for request/response. Paginated auto-generated if omitted. */
  readonly dto?: {
    response?: Type;   create?:  Type;
    paginated?: Type;  update?:  Type;  replace?: Type;
  };

  /** Defaults to [List, Read, Create, Update, Delete] */
  readonly operations?: readonly ResourceOperationName[];

  /** Auto-applied to List + Read via @CrudJoin */
  readonly joins?: readonly ResourceJoinClause[];

  /** Applied via @UseHooks at the controller level */
  readonly hooks?: readonly Type[];

  /** Custom handler per operation (overrides the defaults) */
  readonly handlers?: {
    list?: Type;   read?: Type;
    create?: Type; update?: Type;
    replace?: Type;delete?: Type;
  };

  /** Extra DI providers — handlers/hooks auto-registered */
  readonly providers?: readonly Provider[];

  /** Full override escape hatch */
  readonly overrides?: ResourceOverrides;
}

The three tiers of configuration

TIER 1 — minimal 
defineResource({
  key: 'pet',
  entity: PetEntity,
});
// Full CRUD, no DTO validation,
// paginated DTO auto-gen'd
TIER 2 — typical 
defineResource({
  key: 'pet',
  entity: PetEntity,
  dto: { response, create, update },
  joins: [{ relation: 'owner' }],
  hooks: [OwnerScopeHook],
  handlers: { create: PetCreateHandler },
  providers: [PetCreateHandler, OwnerScopeHook],
});
TIER 3 — full override 
defineResource({
  key: 'pet', entity: PetEntity,
  dto: { response: PetResponseDto },
  overrides: {
    controller: {
      path: 'animals',                   // custom URL
      tags: ['Zoo', 'Internal'],           // custom Swagger tags
      bearerAuth: false,                 // remove default @ApiBearerAuth
      resolver: CustomResolver,           // custom op resolver
      adapter: CustomAdapterProvider,     // custom data adapter
      transactional: true,               // txn wrap all writes
      extraDecorators: [ApiSecurity('api-key')],
      response: { paginated: CustomPaginatedDto },
    },
    operations: {
      list: { transactional: false,
              extraDecorators: [ApiQuery({ name: 'filter' })] },
      create: { command: CustomCreateCommand,
                path: 'new' },
    },
  },
});

How defineResource() assembles the config

1 · Input

  • key: 'pet'
  • entity: PetEntity
  • dto.response
  • joins, hooks, handlers
  • providers, overrides
→

2 · Derive

  • path = pluralize(key)
  • tags = [humanizePlural(key)]
  • paginated = createPaginatedDto(response)
  • resolver = CrudOperationResolver
  • bearerAuth = true
↓

3 · Merge overrides

  • Controller-level overrides win over derived
  • Per-op overrides override default query/command
  • Extra decorators appended after generated
→

4 · Output

  • { crud: { controller, operations }, providers }
  • Same shape as RocketsResourceConfig
  • Fed straight into RocketsCoreModule.forRoot({ resources })

Override escape hatches

Every default defineResource() picks can be overridden without dropping back to the hand-written factory.

OverrideDefaultUse when…
controller.pathpluralize(key)key doesn't pluralize cleanly (petVaccination → pet-vaccinations)
controller.tags[humanizePluralKey(key)]custom Swagger grouping
controller.bearerAuthtruepublic endpoints (false)
controller.resolverCrudOperationResolvercustom operation dispatch
controller.adapterinferred from keyread-only adapters, remote data sources
controller.transactionalupstream defaulttoggle TXN for all mutating ops
controller.extraDecorators—custom class-level decorators (@ApiSecurity, guards)
controller.response.paginatedauto-generatedhand-written paginated DTO with extra fields
operations[op].queryCrudListQuery/CrudReadQuerycustom query class
operations[op].commandCrudCreateCommand/etccustom command class
operations[op].transactionalupstream defaultper-operation TXN toggle
operations[op].extraDecorators—per-route Swagger metadata, guards
operations[op].hooks—hook applied to a single op (not class-wide)
operations[op].path—non-standard route segment
Rule of thumb. If an override feels like "I'm reconstructing the hand-written factory," prefer the hand-written one — it's still a supported code path via RocketsResourceConfig.
